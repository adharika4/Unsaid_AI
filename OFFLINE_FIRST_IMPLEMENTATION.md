# Offline-First PWA Implementation with RunAnywhere SDK

## Overview

Your Unsaid AI app is now fully offline-first using the RunAnywhere SDK for local AI inference. The app implements a complete PWA (Progressive Web App) with automatic model management and offline functionality.

---

## Architecture

```
User opens app
  ↓
Service Worker loads from cache (instant)
  ↓
useOfflineModels hook initializes
  ↓
SDK checks localStorage for downloaded models
  ↓
Auto-loads previously downloaded models (RunAnywhere.loadModel)
  ↓
App is ready for offline use
  ↓
If models not downloaded: Show download buttons
  ↓
User downloads models on WiFi
  ↓
Models stored in OPFS (Origin Private File System)
  ↓
Models persist forever - no re-download needed
```

## How It Works

### First Time User (Needs Internet)

1. User opens app → Service Worker caches assets (JS, CSS, HTML)
2. useOfflineModels hook initializes RunAnywhere SDK
3. SDK checks localStorage - no models found
4. OfflineModelManager component shows download buttons
5. User downloads 3 models on WiFi (Phi-2, Whisper, Piper)
6. Models stored in OPFS with metadata in localStorage
7. Models are auto-loaded after download

### Returning User (No Internet Needed)

1. User opens app → Service Worker serves from cache (instant)
2. useOfflineModels hook initializes SDK
3. Checks localStorage → finds previously downloaded models
4. Auto-loads models from OPFS using `RunAnywhere.loadModel()`
5. App loads models in background without blocking UI
6. User can immediately start using the app
7. All voice features work 100% offline

### What Happens When Offline Without Models Downloaded

- User is shown offline warning
- Download button is disabled (no internet to download)
- Shows: "Please connect to WiFi to download models"

## Technical Details

### Model Management

**Models:**
- `smollm2-360m` - LLM for text generation (500MB)
- `sherpa-onnx-whisper-tiny.en` - Speech-to-Text (390MB)
- `vits-piper-en_US-lessac-medium` - Text-to-Speech (180MB)

**Storage:**
- Models stored in OPFS (Origin Private File System)
- Metadata stored in localStorage for quick checks
- 30-day cache for model downloads from HuggingFace

**API Used:**
```javascript
// Initialize
await RunAnywhere.initialize({ environment: SDKEnvironment.Production })
await LlamaCPP.register()  // For LLM
await ONNX.register()      // For STT/TTS

// Check if downloaded
const isDownloaded = await RunAnywhere.isModelDownloaded(modelId)

// Download
await RunAnywhere.downloadModel(modelId, (progress) => {
  // progress is 0-1, convert to percentage
})

// Load into memory
await RunAnywhere.loadModel(modelId, { coexist: true })

// Delete
await RunAnywhere.deleteModel(modelId)
```

### PWA Configuration

**Service Worker:**
- Auto-registered via vite-plugin-pwa
- Caches all static assets (JS, CSS, HTML, WASM)
- Updates automatically in background
- Workbox handles intelligent caching

**Manifest:**
- App name: "Unsaid AI Mental Health Companion"
- Display: standalone (full-screen app)
- Theme color: dark slate (#0f172a)
- Icons included (SVG with microphone emoji)

**Headers for Multi-Threading:**
- `Cross-Origin-Opener-Policy: same-origin`
- `Cross-Origin-Embedder-Policy: credentialless`
- Enables faster WASM execution with SharedArrayBuffer

## State Flow

```javascript
useOfflineModels returns:
{
  models: {                      // Model definitions with state
    llm: { name, id, downloaded, loaded, progress, ... },
    stt: { ... },
    tts: { ... }
  },
  isDownloading: boolean,        // Currently downloading a model
  currentDownload: modelKey,     // Which model is downloading
  totalStorageUsed: number,      // MB used by downloaded models
  error: string,                 // Error message if any
  downloadModel: function,       // Start downloading
  deleteModel: function,         // Delete a model
  allModelsDownloaded: boolean,  // All 3 models downloaded
  downloadedModelsCount: number, // How many downloaded (0-3)
  isOnline: boolean,             // Online/offline status
  isLoadingModels: boolean,      // Loading models on startup
  allModelsLoaded: boolean,      // All models loaded into memory
}
```

## Troubleshooting

### Models not loading on startup
- Check console for errors
- Verify localStorage has model metadata
- Check OPFS storage in DevTools

### Service Worker not registering
- Check that app is HTTPS (or localhost)
- Check browser console for registration errors
- Verify vite-plugin-pwa is in node_modules

### Large download sizes
- Models are ~1GB total
- Requires WiFi for initial download
- Recommend users download on fast WiFi

### App not working offline after download
- Verify models are in OPFS (DevTools → Application → Storage)
- Check that `isLoadingModels` becomes false
- Verify all 3 models show as `downloaded: true`
